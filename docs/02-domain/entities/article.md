# Article

> üü¢ implemented ¬∑ Area: Knowledge Base ¬∑ Implementation order: 7

## Purpose

A knowledge-base document (markdown): a procedure, troubleshooting note or runbook. The internal
documentation pillar of lazyit ([[vision]]). Started as a deliberately **simple wiki** for small
teams ([[0021-knowledge-base-design]]); [[0042-article-versioning-and-linking]] added the first
depth ‚Äî **append-only versioning** ([[article-version]]), **asset/application linking**
([[article-link]]), and **content indexed in search** ([[0035-search-architecture]]). Tags remain
deferred.

## Relationships

- **grouped by** one [[article-category]] (`categoryId`, **required** FK, `onDelete: Restrict`).
  ‚Üí **Evolving** ([[0059-kb-folders-links-and-import]]): the flat `ArticleCategory` becomes the
  hierarchical [[folder]] (self-ref `parentId`), and this required one-category-per-article FK becomes
  the **one home [[folder]] per article**. The home folder is also the **access boundary**
  ([[0060-kb-folder-access-control]]).
- **authored by** one [[user]] (`authorId`, **required** FK, `onDelete: Restrict`) ‚Äî the creator.
- **last edited by** an optional [[user]] (`lastEditedById`, `onDelete: SetNull`).
- **versioned by** [[article-version]] ‚Äî üü¢ **implemented** ([[0042-article-versioning-and-linking]]):
  an append-only snapshot is written on every create/edit that changes a versioned field, in the same
  transaction as the article write. An article with version history can't be hard-deleted (Restrict).
- **linked to** [[asset]] / [[application]] via [[article-link]] ‚Äî üü¢ **implemented** ([[0042]]):
  each link targets an asset XOR an application ("the runbook for THIS server/app").
- **references / referenced by** other [[article]]s via `[[slug]]` wiki-links ‚Äî ‚ö™ **planned**
  ([[0059-kb-folders-links-and-import]]): a materialized [[article-wiki-link]] edge is rebuilt on
  every save (source ‚Üí `targetSlug`, `resolvedTargetId` nullable), powering **backlinks** + fast
  resolution. This is **distinct** from the IT-native [[article-link]] (article‚Üîasset/application).
- **aliased into** other [[folder]]s via [[article-alias]] ‚Äî ‚ö™ **planned** ([[0059]]): a nav-only
  symlink so one article can appear under several folders. Aliases are presentation only and **never
  widen access** ([[0060-kb-folder-access-control]]).

## Business rules

- **Slug** is unique, short and memorable (`[a-z0-9-]`, ‚â§60 chars). Autogenerated from the title
  (`slugify` in `@lazyit/shared`) but **editable**. A collision returns `409` today; [[0059-kb-folders-links-and-import]] ¬ß3 resolves this ‚Äî on collision the slug will **auto-suffix** (`-2`, `-3`, ‚Ä¶) (planned).
- **Status** is `DRAFT` or `PUBLISHED`. A `DRAFT` is **private to its author**; `PUBLISHED` is
  visible to everyone. Transitions go through `publish`/`unpublish` endpoints ‚Äî **`PATCH` never
  changes status**.
- **`publishedAt`** is set on the **first** publish and **never cleared** (unpublishing keeps it,
  recording "was published at least once").
- **Folder-inherited access** (‚ö™ planned, [[0060-kb-folder-access-control]]): an article's
  visibility is the home [[folder]]'s access scope **composed with** the existing draft-visibility
  rule above ‚Äî both must pass. The `article:read` capability still gates whether you can read at all;
  the folder ACL is an orthogonal **data-scoping** axis that gates *which* articles you see (a bounded
  carve-out from the per-record-ACL rejection of [[0040-rbac-roles]]/[[0046-roles-permissions-v2]] ‚Äî
  access attaches to a named **folder**, not to individual rows). A folder-hidden article returns
  **404, not 403** (existence-hiding, reusing [[0022-draft-visibility-auth-shim]]); **ADMIN** keeps
  god-mode over the KB (INV-8). You can never alias/share an article you cannot yourself access
  (no-escalation ‚Äî proposed **INV-9**).
- **Author-only writes.** Only the author may edit, delete, publish or unpublish. The caller is
  identified by the `X-User-Id` shim until real auth lands ([[0022-draft-visibility-auth-shim]]);
  on create, `authorId` is taken from the caller, **never** the body.
- **Import** (`POST /articles/import`) accepts `.md`, `.txt`, `.docx`, extracts **markdown only**,
  and does **not** store the original file. `.pdf`/`.html`/`.odt` are deferred.
- **Versioning** ([[0042-article-versioning-and-linking]]): create/import write [[article-version]]
  **v1**; an edit that changes `title`/`content`/`excerpt` and a `publish`/`unpublish` (status
  change) append the next version ‚Äî all **in the same transaction** as the article write, so the
  prior body is never lost. A metadata-only or no-op `PATCH` writes no version.
- **Search** ([[0035-search-architecture]] / [[0042]]): a PUBLISHED article is indexed in
  Meilisearch including its **`content`** (runbook bodies are findable). A DRAFT is never indexed
  (author-private), so its content can't leak via search.
- Soft delete ([[0006-soft-delete-and-auditing]]); reads filter `deletedAt: null`.

> [!warning] Auth is a temporary shim ‚Äî endpoints are insecure
> Authorization (draft privacy, author-only writes) is enforced in the service, but the caller is
> just an `X-User-Id` header anyone can set. These endpoints are **unauthenticated/dev-only** until
> an IdP lands ([[0016-auth-strategy-deferred]], [[0022-draft-visibility-auth-shim]]).

> [!warning] Known debt ‚Äî unvalidated `metadata`
> `metadata` (jsonb) accepts **any JSON object** (`z.record(z.string(), z.unknown())`), exactly
> like `Asset.specs` ([[0007-flexible-asset-specs-jsonb]]). Typed per-use validation is deferred.

> [!note] `.docx` conversion uses a deprecated mammoth API
> `.docx ‚Üí markdown` uses `mammoth.convertToMarkdown`, which still ships in mammoth 1.x but is
> absent from its published types (declared locally). If a future mammoth removes it, switch to
> `convertToHtml` + an HTML‚Üímarkdown step. See `apps/api/src/articles/article-import.ts`.

## Conventions

- **ID:** `cuid()` ([[0005-id-strategy]]).
- **Timestamps / soft delete:** `createdAt`, `updatedAt`, `deletedAt`.

## Endpoints

`apps/api/src/articles/` (`ArticlesModule`). The `X-User-Id` header is **optional on reads**
(reveals the caller's own drafts) and **required on writes** (`400` if absent;
[[0022-draft-visibility-auth-shim]]). Documented via Swagger ([[0018-api-documentation-swagger]]).

- `GET /articles` ‚Äî list (excludes soft-deleted; drafts only for their author). Filters
  `?categoryId=&authorId=&status=&q=` (`q` = case-insensitive substring on title/excerpt) plus
  `?linked=only` (keep only articles with ‚â•1 [[article-link]]) and `?linkedTo=asset|application`
  (narrow that to a target kind) ‚Äî both allowlisted, an unknown value ‚Üí `400` ([[0042-article-versioning-and-linking]]).
  The lean list item adds two card-UI affordances computed in the query (no body load, no N+1):
  `linkCount` (relation `_count` of links; `>0` ‚áî linked) and `readingMinutes` (the maintained
  metric). The owner is already exposed as `authorId`.
- `GET /articles/by-slug/:slug` ¬∑ `GET /articles/:id` ‚Äî a draft requested by a non-author returns
  `404` (never `403`, so existence isn't leaked).
- `POST /articles` ‚Äî create (author = `X-User-Id`).
- `POST /articles/import` ‚Äî multipart `.md`/`.txt`/`.docx` import (`MAX_IMPORT_SIZE_MB`, default 5;
  oversize/parse error ‚Üí `400`).
- `PATCH /articles/:id` ‚Äî partial update (author only; never changes status).
- `POST /articles/:id/publish` ¬∑ `POST /articles/:id/unpublish` ‚Äî status transitions (author only).
- `DELETE /articles/:id` ‚Äî soft delete (author only).
- `POST /articles/:id/restore` ‚Äî restore (clears `deletedAt`; author only). ADMIN-gated route; can
  `409` if a live article took the freed slug meanwhile ([[0041-soft-delete-reuse-and-restore]]).

Versioning + linking ([[0042-article-versioning-and-linking]]):

- `GET /articles/:id/versions` ‚Äî paginated version history ([[0030-list-pagination-contract]]),
  newest first; `GET /articles/:id/versions/:version` ‚Äî one version. Drafts visible only to author.
- `POST /articles/:id/links` ‚Äî link to an asset XOR application (author only, `ADMIN`/`MEMBER`);
  `DELETE /articles/:id/links/:linkId` ‚Äî unlink; `GET /articles/:id/links` ‚Äî list links.
- `GET /assets/:id/articles` ‚Äî reverse: PUBLISHED articles linked to an asset (lean list shape).

For a non-author write on a **published** article the API returns `403`; on a **draft**, `404`.
